# 미드저니 V7 파라미터 레퍼런스

## 기본 파라미터

| 파라미터 | 문법 | 범위 | 기본값 | 설명 |
|----------|------|------|--------|------|
| Aspect Ratio | `--ar W:H` | 자유 | 1:1 | 이미지 종횡비 |
| Stylize | `--stylize N` 또는 `--s N` | 0-1000 | 100 | 미드저니 미적 개입 정도 |
| Chaos | `--chaos N` | 0-100 | 0 | 결과 다양성 |
| Weird | `--weird N` | 0-3000 | 0 | 독특함/기이함 |
| Quality | `--quality N` 또는 `--q N` | 0.25, 0.5, 1 | 1 | 렌더링 품질 (V7: 첫 4장에만 영향) |
| Style | `--style X` | raw/expressive/scenic | (없음) | 전반적 스타일 방향 |
| Seed | `--seed N` | 0-4294967295 | 랜덤 | 재현 가능한 결과 |
| Stop | `--stop N` | 10-100 | 100 | 생성 중단 퍼센트 **(V7에서 deprecated)** |
| Tile | `--tile` | flag | off | 타일링 패턴 생성 |
| No | `--no X` | 텍스트 | (없음) | 제외할 요소 |
| Version | `--v N` | 7 (최신) | 7 | 모델 버전 |

## 레퍼런스 파라미터

### V6 레퍼런스

| 파라미터 | 문법 | 범위 | 기본값 | 설명 |
|----------|------|------|--------|------|
| Character Ref | `--cref [URL]` | - | - | 캐릭터 외형(얼굴, 체형, 헤어, 의상) 유지 |
| Style Ref | `--sref [URL]` | - | - | 화풍/색감/질감 유지 |
| Character Weight | `--cw N` | 0-100 | 100 | 캐릭터 참조 강도 |
| Style Weight | `--sw N` | 0-1000 | 100 | 스타일 참조 강도 |
| Style Version | `--sv N` | 1-6 | 6 | 스타일 참조 해석 버전 |

### V7 레퍼런스 (Omni Reference)

| 파라미터 | 문법 | 범위 | 기본값 | 설명 |
|----------|------|------|--------|------|
| Omni Ref | `--oref [URL]` | - | - | 통합 레퍼런스 (사람/물체/동물 모두 가능) |
| Omni Weight | `--ow N` | 0-1000 | 100 | Omni Reference 가중치 |

> **V7 --oref 주의사항**: GPU 비용 2배. Inpainting, Outpainting, Vary Region, Fast Mode, Draft Mode, `--q 4`와 호환 불가.

### --cw (Character Weight) 상세

| 값 | 효과 | 사용 시점 |
|----|------|----------|
| 0 | **얼굴만** 유지, 의상/헤어 자유 변경 | 의상 변경, 코스튬 차이 |
| 50 | 얼굴 + 대략적 외형 유지 | 부분적 변형 허용 |
| 80 | 대부분의 외형 유지, 약간의 유연성 | **포즈 변경 시 추천** |
| 100 (기본) | 얼굴+헤어+의상 **최대** 유지 | 캐릭터 동일성 최우선 |

### --ow (Omni Weight) 상세

| 값 | 효과 | 사용 시점 |
|----|------|----------|
| 25~50 | 미묘한 영향, 스타일 변환 허용 | 다른 화풍으로 변환 |
| 100 (기본) | 균형 잡힌 참조 | 범용 |
| 200~300 | 강한 디테일 보존 | **캐릭터 일관성 + 포즈 변경 시 추천** |
| 400~1000 | 극도로 강한 참조 | 세밀한 복제 (예측 불가 위험) |

### 레퍼런스 조합 패턴

| 목적 | 조합 | 설명 |
|------|------|------|
| 캐릭터 유지 + 포즈 변경 | `--cref + --sref` (같은 이미지) | 외모 + 그림체 동시 유지 |
| 정확한 포즈 지정 | Image Prompt(포즈 사진) + `--cref` | 포즈 참조 + 캐릭터 유지 |
| 의상 변경 | `--cref --cw 0` + `--sref` | 얼굴만 유지, 나머지 자유 |
| V7 최적 | `--oref --ow 200` + `--sref` | 통합 참조 + 스타일 고정 |

### 레퍼런스 사용 시 주의사항
- `--cref`만 사용하면 **그림체가 달라질 수 있음** → `--sref`를 같은 이미지로 함께 사용
- `--stylize` 값이 높으면 `--cref`/`--sref` 효과 약화 → 캐릭터 일관성이 중요하면 `--s 50~150`
- 웹 UI에서 같은 이미지를 **Shift+클릭**으로 Character Ref + Style Ref 동시 설정 가능
- **Editor(Edit 버튼) 모드에서는 --cref, --sref, --oref, --ar 등 대부분의 파라미터 미지원** → Create 페이지 사용 필요

## 파라미터 상세 설명

### Stylize (--s)
미드저니의 미적 개입 정도를 제어합니다.

| 값 | 효과 | 권장 용도 |
|----|------|----------|
| 0 | 프롬프트만 충실히 반영 | 기술 도면, 정확한 재현 |
| 50 | 약간의 미화 | 포토리얼, UI 아이콘 |
| 100 (기본) | 균형 잡힌 미화 | 범용 |
| 200 | 적당한 미적 강화 | 컨셉아트, 캐릭터 |
| 300 | 뚜렷한 미화 | 일러스트, 환경 |
| 500 | 강한 미드저니 스타일 | 예술적 작품 |
| 750 | 매우 강한 스타일 | 추상, 실험적 |
| 1000 | 극대화 | 극도로 예술적 |

### Chaos (--chaos)
생성 결과의 다양성을 제어합니다.

| 값 | 효과 | 권장 용도 |
|----|------|----------|
| 0 (기본) | 일관된 결과 | 최종 결과물 |
| 10-20 | 미세한 변형 | 약간의 탐색 |
| 30-50 | 중간 변형 | 영감 찾기 |
| 60-80 | 큰 변형 | 다양한 방향 탐색 |
| 100 | 극도의 다양성 | 완전히 새로운 해석 |

### Style (--style)
전반적인 렌더링 방향을 설정합니다.

| 옵션 | 효과 | 권장 용도 |
|------|------|----------|
| (없음) | 기본 미드저니 스타일 | 대부분의 경우 |
| raw | 최소 미화, 프롬프트 충실 | 포토리얼, 정확한 재현 |
| expressive | 예술적 표현 극대화 | 아트워크, 일러스트 |
| scenic | 풍경/환경 최적화 | 배경, 풍경 |

### Weird (--weird)
독특하고 예상치 못한 요소를 추가합니다.

| 값 | 효과 |
|----|------|
| 0 (기본) | 일반적 결과 |
| 100-300 | 약간의 독특함 |
| 500-1000 | 뚜렷한 기이함 |
| 1500-3000 | 초현실적, 극도로 실험적 |

### Quality (--q)
렌더링에 할당되는 GPU 시간을 제어합니다.

> **V7 주의**: Quality 파라미터는 V7에서 **첫 번째 4장 생성에만** 영향을 미칩니다. 이후 Variation, Inpainting, Upscale에는 적용되지 않습니다.

| 값 | 효과 |
|----|------|
| 0.25 | 빠른 생성, 낮은 디테일 (4배 빠름) |
| 0.5 | 빠른 생성, 중간 디테일 (2배 빠름) |
| 1 (기본) | 표준 품질 |

## 카테고리별 프리셋 조합

### 일반 프리셋

| 프리셋 | 파라미터 |
|--------|---------|
| 포토리얼 인물 | `--ar 3:4 --style raw --stylize 80 --chaos 5` |
| 시네마틱 | `--ar 21:9 --style raw --stylize 120` |
| 판타지 컨셉아트 | `--ar 16:9 --stylize 300 --chaos 15` |
| 캐릭터 디자인 | `--ar 2:3 --stylize 200` |
| 일러스트 | `--ar 3:4 --stylize 400` |
| 아이콘/UI | `--ar 1:1 --stylize 50` |

### 게임 아트 프리셋

| 프리셋 | 파라미터 |
|--------|---------|
| 캐릭터 시트 | `--ar 16:9 --stylize 150 --chaos 5` |
| 환경 컨셉 | `--ar 16:9 --stylize 300 --style scenic` |
| 무기/아이템 | `--ar 1:1 --stylize 150` |
| 스프라이트 시트 | `--ar 16:9 --stylize 50` |
| 타일셋 | `--ar 1:1 --tile --stylize 50` |
| 보스 디자인 | `--ar 2:3 --stylize 250 --chaos 20` |

### 실험/탐색 프리셋

| 프리셋 | 파라미터 |
|--------|---------|
| 영감 탐색 | `--chaos 50 --stylize 200` |
| 초현실적 | `--weird 2000 --chaos 30 --stylize 500` |
| 캐릭터 일관성 (V6) | `--cref [URL] --cw 100 --sref [URL] --sw 100 --stylize 100` |
| 캐릭터 일관성 (V7) | `--oref [URL] --ow 200 --sref [URL] --stylize 100` |
| 스타일 전이 | `--sref [URL] --sw 100 --stylize 150` |
| 타일 패턴 | `--tile --stylize 100 --ar 1:1` |

## 팁

### 파라미터 조합 원칙
1. `--style raw` + 낮은 `--stylize`: 최대 프롬프트 충실도
2. 높은 `--stylize` + 높은 `--chaos`: 예술적 탐색
3. `--weird` > 1000: 반드시 `--chaos` 20-50과 함께 사용
4. `--cref`와 `--sref`: 동시 사용 가능하나 강도 조절 필요

### 흔한 실수
- `--ar`에 소수점 사용 (X) → 정수 비율만 (O)
- `--stylize 0`으로 너무 낮게 설정 → 결과물이 밋밋해짐
- 파라미터를 너무 많이 사용 → 핵심 2-3개만 사용 권장
- `--no`를 과도하게 사용 → 결과 품질 저하
- Image Prompt에 이미지 넣고 텍스트로 포즈 변경 시도 → --cref 사용해야 함
- Editor(Edit) 모드에서 --ar, --cref 등 사용 시도 → Create 페이지에서만 지원

### 웹 UI 파라미터 지원 참고

| 모드 | 지원 파라미터 | 미지원 |
|------|-------------|--------|
| **Create 페이지 (Imagine 바)** | 모든 파라미터 | - |
| **Editor (Edit 버튼)** | 텍스트 프롬프트만 | --ar, --stylize, --cref, --sref, --oref 등 대부분 |

## V7 신규 파라미터

### Personalization (--p)
V7에서 기본적으로 활성화된 개인화 기능. 사용자의 미적 선호도를 학습하여 결과물에 반영합니다.

| 파라미터 | 문법 | 설명 |
|----------|------|------|
| Personalization | `--p` 또는 `--personalization` | 개인화 적용 (V7 기본 활성화) |

> **설정**: 5분 내 이미지 평가로 빠르게 개인화 프로필 생성 가능 (V6 대비 ~200장 평가 불필요).

### Draft Mode
10배 빠른 생성 속도, GPU 비용 50% 절감. 아이디어 빠른 탐색에 최적.

| 설정 | 효과 |
|------|------|
| Draft Mode ON | 10x 빠른 생성, 비용 절반. 대화식 수정 가능 ("고양이를 부엉이로 바꿔") |
| Draft Mode OFF (기본) | 표준 속도, 표준 비용 |

> **활용 워크플로우**: Draft Mode로 빠르게 탐색 → 마음에 드는 결과 확인 → Fast/Turbo 모드로 고품질 최종 생성

### Niji 7 (--niji)
2026년 1월 출시. 애니메이션/동양 일러스트 특화 모델.

| 파라미터 | 문법 | 설명 |
|----------|------|------|
| Niji | `--niji 7` | 애니/만화 스타일 특화 모델 사용 |

> **제한사항**: Niji 7은 현재 `--cref` (Character Reference)와 `--oref` (Omni Reference) **미지원**. `--sref` (Style Reference)는 사용 가능.

### Style Version (--sv)
Style Reference (`--sref`)의 해석 방식을 제어합니다.

| 파라미터 | 문법 | 범위 | 기본값 | 설명 |
|----------|------|------|--------|------|
| Style Version | `--sv N` | 1-6 | 6 | 스타일 참조 해석 버전 |

## 프롬프트 토큰 제한

| 버전 | 토큰 제한 | 권장 단어 수 |
|------|----------|-------------|
| V6 이하 | 75 토큰 | ~30단어 |
| V6.1, V7 | **150 토큰** | ~40~80단어 |

> **참고**: 토큰 ≠ 단어. 복잡하거나 비일반적인 단어는 여러 토큰을 소비할 수 있습니다. V7은 짧은 프롬프트로도 의도를 잘 이해하므로 **간결한 프롬프트를 권장**합니다.
>
> **Multi-Prompt**: `::` 구분자를 사용하면 각 세그먼트가 별도의 74토큰 제한을 가져, 복잡한 아이디어를 더 자세히 표현할 수 있습니다.
