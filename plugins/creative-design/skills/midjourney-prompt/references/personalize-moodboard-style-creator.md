# Personalize, Moodboards & Style Creator 가이드

미드저니 웹 UI(midjourney.com) 전용 3대 스타일 제어 기능의 상세 가이드.
이 3가지를 조합하면 **일관된 미적 방향성**을 유지하면서 프롬프트를 생성할 수 있다.

---

## 1. Personalization (개인화)

### 개념

미드저니가 사용자의 **미적 선호도**를 학습하여, 프롬프트 해석 시 개인 취향에 맞는 방향으로 결과를 생성하는 기능.
V7에서 **기본 활성화**되며, 최초 ~200장 이미지 평가(약 5분)가 V7 사용의 전제조건이다.

### 프로필 타입 3종

| 프로필 타입 | 학습 방법 | 용도 |
|------------|----------|------|
| **Global Profile** | Pair Rankings(쌍 비교) + Explore 페이지 좋아요 | 범용 기본 취향 |
| **Ranked Profile** | 별도의 Pair Rankings 세션 | 특정 스타일 전용 (예: 사진 vs 일러스트 vs 호러) |
| **Moodboard Profile** | 업로드한 이미지 컬렉션 기반 | 프로젝트/클라이언트별 미학 |

### 학습 방법

1. **Pair Rankings** (핵심): `midjourney.com/personalize` → Tasks 탭에서 2장 비교 후 선호 이미지 선택
2. **Explore 좋아요**: Explore 페이지에서 커뮤니티 이미지에 좋아요 → Global Profile에 반영
3. **Moodboard**: 이미지 컬렉션을 직접 큐레이션 (아래 Moodboards 섹션 참조)

#### 학습 수준별 효과

| 평가 수 | 효과 |
|---------|------|
| ~200장 | 최소 활성화 (V7 접근 해제) |
| ~2,000장 | 거의 완벽한 캘리브레이션 |
| 5~10 라운드 | 미적 방향 안정화 |
| 10~15 라운드 | 디테일 수준 정교화 |
| 15+ 라운드 | 미세한 변화만 |

### 웹 UI 조작

- **Imagine 바 옆 Personalize 버튼**: 클릭으로 ON/OFF 토글
- **드롭다운 화살표**: 활성 프로필 전환 (Global, Ranked, Moodboard 중 선택)
- ON이면 모든 프롬프트에 자동으로 기본 프로필의 `--p` 코드가 적용됨
- 수동으로 `--p` 또는 `--p [코드]`를 직접 입력해도 동일

### `--p` 파라미터 사용법

| 문법 | 효과 |
|------|------|
| `--p` | 기본 프로필 적용 |
| `--p abc1234` | 특정 프로필 코드 적용 |
| `--p abc1234 xyz5678` | 다중 프로필 블렌딩 |

### `--stylize`와의 연동

`--stylize` 값이 Personalization 강도에 영향:
- 낮은 `--stylize` (0~100) → 개인화 효과 약함
- 중간 `--stylize` (200~300) → 눈에 띄는 개인화
- 높은 `--stylize` (500+) → 개인화 + 미드저니 미학이 강하게 결합

### 공유 코드 시스템

- **코드 확인**: `midjourney.com/personalize` → 프로필 호버 → 코드 복사 버튼
- **타인 코드 사용**: `--p theircode` (누구나 사용 가능)
- **제한**: 프로필 학습(추가 랭킹)은 원본 생성자만 가능
- 프로필은 랭킹 추가 시 점진적으로 진화 → 공유 코드도 함께 업데이트됨

### 프롬프트 전략 (Personalize ON일 때)

- **"무엇을(What)"에 집중, "어떻게(How)"는 줄이기** — 스타일은 프로필이 처리
- 스타일 키워드를 과도하게 넣으면 프로필과 충돌할 수 있음
- 별도 미학이 필요한 프로젝트에는 **Ranked Profile을 새로 생성**하여 분리
- `--sref`와 `--p`는 **스택 가능** — `--sw`로 sref 가중치를 별도 제어

## 2. Moodboards (무드보드)

### 개념

사용자가 직접 큐레이션한 이미지 컬렉션으로 미드저니를 학습시키는 기능.
Pair Rankings 기반의 Personalization과 달리, **이미지를 직접 선택**하여 정확한 미적 방향을 지정한다.
**브랜드 일관성, 클라이언트 프로젝트, 팀 워크플로우**에 최적.

### 생성 워크플로우

1. `midjourney.com/personalize` 또는 좌측 사이드바 Personalize 아이콘 클릭
2. **"Create Moodboard"** 클릭
3. 이미지 추가 (3가지 방법):
   - **직접 업로드**: 로컬 이미지 드래그앤드롭
   - **웹 링크**: 이미지 URL 붙여넣기
   - **갤러리에서 가져오기**: 과거 미드저니 생성 이미지 선택
4. 무드보드 이름 지정 (예: "Moody Editorial", "Pastel Brand")
5. 선택적으로 기본 프로필로 설정

**용량**: 무드보드당 최대 **100장**.

### 2가지 코드 체계 (핵심)

| 코드 타입 | 형식 | 예시 | 특성 |
|----------|------|------|------|
| **M-code (동적)** | `--p m[긴 숫자]` | `--p m7267801530722418712` | 무드보드에 실시간 연동. 이미지 추가/삭제 시 결과 변경. 무드보드 삭제 시 무효화 |
| **Snapshot code (정적)** | `--profile [짧은 영문숫자]` | `--profile r1bwkpa` | 특정 시점의 스냅샷. 무드보드 삭제 후에도 영구 유지. 이미지 추가/삭제마다 새 스냅샷 코드 생성 |

> **핵심**: 장기 프로젝트/공유용에는 **Snapshot code** 사용 권장. 무드보드가 삭제되거나 수정되어도 안전하다.

### 프롬프트에서 사용

```
[프롬프트] --profile [snapshot code]
```
또는
```
[프롬프트] --p [m-code]
```

웹 UI에서는 Personalize 토글의 드롭다운에서 기본 무드보드를 선택하면 자동 적용.

### 블렌딩 (다중 조합)

**Moodboard + Moodboard:**
```
[프롬프트] --profile drgmjoi 2jrqbw6
```

**Moodboard + --sref:**
```
[프롬프트] --sref 142710498 --profile drgmjoi
```

**Moodboard + Moodboard + --sref:**
```
[프롬프트] --sref 316835587 938444257 --profile emgo5p1 vdb1o7h
```

> **제한**: Moodboard 코드는 **가중치(`::`) 미지원**. `--sref`는 가중치 가능.

### Moodboard vs Sref vs Personalization 비교

| | Moodboard | --sref | Personalization Profile |
|--|-----------|--------|------------------------|
| **생성 방식** | 이미지 직접 큐레이션 | 숫자 코드 또는 이미지 업로드 | AI 기반 쌍 비교 평가 |
| **스타일 출처** | 사용자가 선택한 이미지 | 기존 코드 또는 참조 이미지 | 미적 선호도 이력 |
| **유연성** | 동적 업데이트 가능 | 코드당 고정 | 평가 추가로 점진 진화 |
| **가중치** | 미지원 | `::` 가중치 지원 | 미지원 |
| **최적 용도** | 브랜드 일관성, 클라이언트 프로젝트 | 실험적 스타일 탐색 | 범용 개인 취향 |

### 베스트 프랙티스

- **5~10장으로 시작** → 명확한 미적 방향 확인 후 확장
- 하나의 무드보드에 **일관된 미학**의 이미지만 수집 (랜덤 수집 X)
- 클라이언트/프로젝트별 **별도 무드보드** 생성
- 장기 프로젝트에는 **Snapshot code** 사용 (M-code는 변동 위험)
- 테스트 시 간단한 프롬프트 + `--chaos 10 --stylize 200`으로 코드 성능 확인
- 여러 기본 프로필을 동시에 설정하여 자동 적용 가능

## 3. Style Creator (스타일 크리에이터)

### 개념

시각적 선호도 선택 과정을 통해 **커스텀 `--sref` 코드**를 생성하는 웹 전용 도구.
더 이상 사용할 수 없는 **Style Tuner(디스코드 `/tune` 명령어)의 후속** 기능.
40억 개 이상의 고유 스타일 생성 가능.

### 워크플로우

1. **좌측 메뉴 → Style Creator** 클릭
2. **프롬프트 입력** (3가지 전략 중 택 1):

| 방법 | 입력 | 최적 용도 |
|------|------|----------|
| **Dot method (추천)** | `.` (마침표만) | 주제에 구애받지 않는 범용 스타일 — 이식성 최대 |
| **구체적 프롬프트** | 전체 설명 텍스트 | 특정 콘텐츠 타입에 최적화된 스타일 |
| **기존 이미지 리파인** | 이미지 호버 → Style Creator | 마음에 드는 생성 결과를 기반으로 발전 |

3. **초기 그리드 확인**: 10장 또는 24장의 서로 다른 미학 이미지 표시
4. **선호 이미지 선택**: 클릭으로 선택 (미선택 이미지도 AI 학습에 반영)
5. **"Refine Style" 반복**: 선택 기반으로 새 그리드 생성 → 반복
6. **세션 종료**: X 또는 "End Session" 클릭 → 커스텀 `--sref` 코드 생성
7. **코드 저장**: 숫자 코드를 기록 (나중에 복구 불가)

### 리파인먼트 라운드 가이드

| 라운드 | 상태 | 설명 |
|--------|------|------|
| 1~5 | 탐색 | 높은 변동, 랜덤해 보일 수 있음 — 정상 |
| 5~10 | 안정화 | 인식 가능한 미적 방향 등장 |
| 10~15 | 정교화 | 미세한 디테일 조정 |
| 15+ | 수렴 | 미미한 변화 — 대부분 여기까지 불필요 |

**실용적 최적 구간: 5~15 라운드**

### 그리드 컨트롤

- **그리드 크기 전환**: 10장 ↔ 24장
- **Skip Grid**: 현재 그리드 건너뛰기 (패널티 없음)
- **Start New Session**: 현재 이미지를 새 기준점으로 리파인먼트 재시작
- **키보드 단축키**: 토글 가능 (빠른 선택용)
- **북마크**: 세션 중 마음에 드는 스타일 저장

### 생성된 코드 사용

```
[프롬프트] --sref [생성된 숫자 코드]
```

**스타일 강도 조절:**
```
[프롬프트] --sref [코드] --sw 50
```

**다중 코드 블렌딩 (가중치 지원):**
```
[프롬프트] --sref 2588022918::3 2302673646::2
```

**Moodboard와 조합:**
```
[프롬프트] --sref 142710498 --profile drgmjoi
```

### `--sv` (Style Version) 주의사항

| 코드 생성 시점 | 기본 --sv | 비고 |
|-------------|---------|------|
| 2025.06.16 이후 (V7 신규) | `--sv 6` | 자동 적용 |
| 2025.06.16 이전 (레거시) | 수동 `--sv 4` 필요 | 시스템이 코드 연대를 감지 못함 |

### Style Creator vs Style Tuner (레거시)

| | Style Creator (현행) | Style Tuner (레거시) |
|--|---------------------|---------------------|
| 상태 | 활성 (웹 전용) | 비활성 (신규 생성 불가) |
| 플랫폼 | midjourney.com | 디스코드 `/tune` |
| 코드 타입 | `--sref` 숫자 코드 | `--style` 코드 |
| 버전 | V7 / `--sv 6` | V5.2 시대 |
| 방식 | 반복 리파인먼트 | 1회 선택 |

### 베스트 프랙티스

- **`.`(dot)으로 시작**하면 가장 이식성 높은 스타일 코드 생성
- 그리드 이미지의 **스타일에 집중** (주제/피사체는 실제 프롬프트와 다를 수 있음)
- 1~2 라운드에서 판단하지 말고 **최소 5 라운드** 진행
- 생성된 코드는 **반드시 메모** (숫자로만 되어 있어 나중에 검색 불가)
- `--sw`(style weight)로 강도 조절 후 → 미세 조정이 안 되면 새 코드 생성
- 장기 프로젝트: **Style Creator 코드 + Moodboard**를 레이어링하면 정교한 미학 제어 가능

## 3가지 기능 조합 전략

### 워크플로우별 추천 조합

| 워크플로우 | 추천 조합 | 설명 |
|-----------|----------|------|
| **개인 프로젝트 (자유 탐색)** | Personalization ON | 기본 취향 자동 반영, 추가 설정 불필요 |
| **특정 미학 프로젝트** | Ranked Profile + --sref | 프로젝트 전용 프로필 + Style Creator 코드 |
| **클라이언트/브랜드 작업** | Moodboard(Snapshot) + --sref | 클라이언트 레퍼런스 기반 무드보드 + 스타일 코드 |
| **팀 협업** | Moodboard(Snapshot 공유) | Snapshot code 공유로 팀 전원 동일 미학 |
| **새 스타일 탐색** | Style Creator → Moodboard 확장 | Style Creator로 방향 찾기 → Moodboard로 고정 |
| **기존 캐릭터 + 일관된 미학** | Moodboard + --oref + --sref | 캐릭터 일관성 + 프로젝트 미학 동시 유지 |

### 우선순위 규칙

여러 스타일 소스가 충돌할 때:
1. `--sref` (Style Reference) → 가장 직접적이고 구체적
2. `--profile` / `--p` (Moodboard/Personalization) → 간접적 미학 가이드
3. 프롬프트 텍스트의 스타일 키워드 → 보조적 역할

> **팁**: `--sref`와 Personalization을 동시에 사용할 때, `--stylize`를 낮추면(50~150) 참조 충실도가 높아진다.

### 실전 프롬프트 예시

**Style Creator 코드 + Moodboard 조합:**
```
a cozy mountain cabin at sunset, warm interior glow --sref 2588022918 --profile r1bwkpa --ar 16:9 --stylize 200
```

**다중 Moodboard 블렌딩:**
```
fantasy warrior standing in ancient ruins --profile drgmjoi 2jrqbw6 --ar 2:3 --stylize 250
```

**Personalization + Sref 스택:**
```
ethereal forest landscape, morning mist --p --sref 938444257 --sw 150 --ar 16:9
```
