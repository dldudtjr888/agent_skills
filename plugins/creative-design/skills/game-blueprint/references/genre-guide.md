# 장르별 Phase 어댑터 가이드

장르가 파악되면 이 가이드를 참조해서 각 Phase의 질문과 템플릿을 조정한다.

---

## RPG / 탐색 게임

탑다운 또는 사이드뷰 RPG, 탐색 중심 게임.

### Phase 0 추가 질문
- 시점? (탑다운 / 사이드뷰 / 아이소메트릭)
- 전투 시스템? (턴제 / 실시간 / 없음)
- 이동 시스템? (타일 기반 / 자유 이동)
- 인벤토리/아이템? (있음 / 없음 / 간소화)

### Phase 1 중점
- 퀘스트 구조 (메인/서브)
- NPC 수가 많을 수 있음 — 핵심 NPC와 배경 NPC 구분
- 분기는 대화 선택지 + 퀘스트 완료 여부로 결정
- 스코어링은 스탯 기반 가능 (힘/지능/카리스마 등)

### Phase 2 중점
- **맵 설계 필수**: walkable, collision, interactable 영역 상세 정의
- HUD: HP/MP/경험치 바, 미니맵, 인벤토리 버튼
- 대화 UI: 초상화 + 대화창 + 선택지
- 전투 UI (있다면): 액션 선택, 적 정보, 턴 표시

### Phase 3 중점
- 스프라이트: 플레이어(걷기 4방향×4프레임, 공격 등), NPC(걷기+대기)
- 타일셋: 맵별 타일셋(실내/실외/던전), 시간대 변형
- 초상화: NPC별 표정 3~5종
- BGM: 맵별 + 전투 + 이벤트

## 비주얼 노벨

텍스트/대화 중심, 선택 기반 분기 게임.

### Phase 0 추가 질문
- 탐색 요소? (순수 대화만 / 포인트앤클릭 / 맵 이동)
- 분기 깊이? (선형 / 약간 분기 / 복잡한 트리)
- 음성? (풀보이스 / 일부 / 없음)

### Phase 1 중점
- **분기 트리가 핵심**: 선택지 → 루트 분기 상세 설계
- 엔딩 수가 많을 수 있음 (5~10+)
- NPC 심층 설계: 호감도 시스템, 개인 루트
- 씬 단위 구성 (챕터 → 씬 → 대화 블록)

### Phase 2 중점
- 대화창 레이아웃: 캐릭터 위치(좌/중/우), 이름 표시, 텍스트 영역
- 선택지 UI: 2~4개 선택, 타이머 여부
- 배경 전환: 페이드/슬라이드/디졸브
- 갤러리/CG 화면 (있다면)
- 세이브/로드 UI

### Phase 3 중점
- **일러스트 배경이 가장 많음**: 장소별 배경 (시간대/날씨 변형)
- 캐릭터 초상화: 표정 5~8종 × 캐릭터 수
- 특수 CG 일러스트 (이벤트 씬)
- UI: 대화창, 이름 태그, 선택지 버튼
- BGM: 분위기별 (일상/긴장/감동/엔딩)

## 퍼즐 게임

매 스테이지 퍼즐을 풀어나가는 게임.

### Phase 0 추가 질문
- 핵심 퍼즐 메카닉? (매칭? 물리? 논리? 단어?)
- 난이도 곡선? (선형 증가 / 물결형 / 자유 선택)
- 스테이지 수? (예상)
- 힌트 시스템?

### Phase 1 중점
- 레벨 진행 구조 (월드 → 스테이지)
- 내러티브가 있다면: 챕터별 테마 (숲 → 바다 → 화산)
- 튜토리얼 흐름 (새 메카닉 소개 타이밍)
- 별점/클리어 등급 조건

### Phase 2 중점
- 레벨 선택 화면 (월드맵? 리스트?)
- 플레이필드 레이아웃 (그리드 크기, 조작 방식)
- 결과 화면 (별점, 점수, 다음 레벨)
- 일시정지 / 힌트 UI
- 상점 (있다면)

### Phase 3 중점
- 퍼즐 피스/블록: 종류별 스프라이트
- 배경: 테마별 (5~10개)
- 이펙트: 매칭, 폭발, 클리어 애니메이션
- UI: 버튼, 별점, 게이지
- SFX: 조작, 매칭, 클리어, 실패

## 시뮬레이션 / 타이쿤

경영, 육성, 전략 시뮬레이션.

### Phase 0 추가 질문
- 시뮬레이션 대상? (회사? 도시? 캐릭터? 생태계?)
- 자원 체계? (돈, 시간, 명성, 체력 등)
- 시간 흐름? (턴제 / 실시간 / 이벤트 기반)
- 실패 조건? (파산? 타임오버? 평판?)

### Phase 1 중점
- **이벤트 체인이 핵심**: 랜덤/조건부 이벤트
- 자원 흐름: 수입/지출/투자 구조
- 마일스톤/목표: 단기(일/주) + 장기(전체)
- 실패 조건과 경고 시스템

### Phase 2 중점
- **대시보드가 핵심 화면**: 자원 게이지, 그래프, 지표
- 타임라인/캘린더 UI
- 이벤트 팝업: 상황 + 선택지
- 보고서/리포트 화면
- 직원/캐릭터 관리 화면 (있다면)

### Phase 3 중점
- 아이콘: 자원, 액션, 상태 아이콘 세트
- UI: 게이지, 그래프, 카드, 버튼
- 환경: 배경 (오피스, 공장, 도시 등)
- 캐릭터: 간소화된 초상화 (이벤트용)
- BGM: 평시/위기/성장

## 액션 / 플랫포머

실시간 조작 중심, 레벨 기반 액션 게임.

### Phase 0 추가 질문
- 시점? (사이드뷰 / 탑다운)
- 이동 감각? (무거운/가벼운/정밀한)
- 공격/능력? (검? 총? 마법? 격투?)
- 레벨 구조? (선형 / 메트로이드바니아 / 로그라이크)

### Phase 1 중점
- 레벨 구조: 월드 → 스테이지 → 보스
- 적 종류: 잡몹/정예/보스 정의
- 파워업/아이템 체계
- 난이도 곡선

### Phase 2 중점
- HUD: HP, 능력 쿨다운, 점수/시간
- 레벨 뷰: 카메라 범위, 스크롤 방식
- 일시정지 화면
- 사망/리트라이 화면
- 보스전 UI (있다면)

### Phase 3 중점
- 플레이어 스프라이트: 걷기/달리기/점프/공격/피격/사망 (프레임 수 많음)
- 적 스프라이트: 종류별 idle/attack/death
- 타일셋: 레벨 테마별
- 이펙트: 공격, 폭발, 파티클
- BGM: 레벨별 + 보스전
- SFX: 점프, 공격, 피격, 사망, 아이템

## 하이브리드 (복합 장르)

여러 장르가 섞인 경우 (예: RPG + 비주얼노벨, 시뮬레이션 + 탐색).

### 접근법
1. **주 장르**를 먼저 파악 (플레이 시간의 60%+ 차지하는 것)
2. **부 장르**의 Phase 중점 항목을 보조로 추가
3. **모드 전환**이 있으면 Phase 2에서 각 모드별 화면을 독립적으로 설계
4. 에셋은 두 장르의 합집합

### 예시: 탑다운 RPG + 비주얼노벨
- Phase 0: RPG 기반 (시점, 이동) + VN 추가 (대화 비중)
- Phase 1: RPG (퀘스트, NPC) + VN (분기 트리, 엔딩 수)
- Phase 2: 탐색 맵 화면 + 대화 화면 **독립 설계**
- Phase 3: 스프라이트 + 타일셋 + 초상화(표정 많이) + 배경
