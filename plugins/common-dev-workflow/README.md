# common-dev-workflow

공통 개발 워크플로우: 태스크 계획/분해/실행, 문서 관리, 코드 분석

## Skills

| Skill | Description |
|-------|-------------|
| task-planner | 요구사항 구체화 및 계획 문서 작성 |
| task-decomposer | 계획을 실행 가능한 태스크로 분해 |
| task-executor | 태스크 순차 실행, 에이전트 위임 |
| project-docs-manager | 프로젝트 문서 관리 |
| code-refactoring-analysis | 5차원 코드 리팩토링 분석 |
| sql-production-analyzer | SQL 쿼리 및 프로덕션 분석 |
| claude-code-pattern-catalog | Claude Code 패턴 카탈로그 |

## Agents

| Agent | Description |
|-------|-------------|
| architect | 시스템 아키텍처 설계 |
| analyst | 요구사항/데이터 분석 |
| explore | 코드베이스 탐색 |
| critic | 비판적 리뷰, 개선점 도출 |
| researcher | 조사/리서치 |
| scientist | 실험 설계, 가설 검증 |
| qa-tester | QA 테스트 전략 |
| refactor-planner | 리팩토링 계획 수립 |
| database-reviewer | DB 스키마/쿼리 리뷰 |
| tradeoff-analyzer | 기술적 트레이드오프 분석 |

## Usage

```bash
/plugin install common-dev-workflow@hibye-plugins
```

모든 도메인 플러그인과 함께 사용 권장.
